변환 중 허상(할루시네이션)을 줄이는 두 장치

CUT (SAR → Optical) 코드에 들어간 Edge λ (lambda_grad) 와 Color λ (lambda_color) 가 실제 연산에서 어떻게 적용되는지를 그림과 애니메이션으로 풀어봅니다. 수식·코드 위치까지 그대로 매칭했습니다.

🎬 한눈에 보기 — "없던 게 생겨나는" 허상을 억제

생성기 G가 SAR을 광학영상으로 바꿀 때, 입력에 없던 건물·도로·색을 멋대로 그려 넣는 일이 생깁니다(허상). Edge λ는 구조 허상을, Color λ는 색 허상을 누릅니다.

0. 두 λ는 "생성기 손실"에 더해지는 항이다

학습 중 생성기 G는 아래 하나의 손실을 줄이도록 업데이트됩니다. 우리가 추가한 두 항은 맨 뒤에 λ로 가중치를 곱해 더해집니다. λ가 클수록 그 규칙을 더 강하게 강제합니다.

LG = LGAN + LNCE + λgrad·Ledge(real_A, fake_B) + λcolor·Lcolor(idt_B, real_B)
GAN: "광학처럼 보이게" NCE: "입력 내용 유지(대조학습)" Edge λ: "구조(윤곽) 유지" → 구조 허상↓ Color λ: "색 통계 유지" → 색 허상↓

코드에서는 models/cut_model.py 의 compute_G_loss() 끝부분입니다:

# models/cut_model.py — compute_G_loss()
self.loss_G = self.loss_G_GAN + loss_NCE_both
if self.opt.lambda_grad > 0.0:
    self.loss_G_grad  = gradient_loss(self.real_A, self.fake_B) * self.opt.lambda_grad
    self.loss_G       = self.loss_G + self.loss_G_grad        # ← Edge λ 항
if self.opt.lambda_color > 0.0 and self.opt.nce_idt:
    self.loss_G_color = color_moment_loss(self.idt_B, self.real_B) * self.opt.lambda_color
    self.loss_G       = self.loss_G + self.loss_G_color       # ← Color λ 항
return self.loss_G
📌 핵심 짝꿍: Edge 는 입력 real_A ↔ 출력 fake_B 를 비교하고, Color 는 idt_B(=G(real_B)) ↔ real_B 를 비교합니다. 그래서 Color λ는 nce_idt 가 켜져 있을 때(=idt_B가 존재할 때)만 동작합니다.

1. "허상"이 왜 생길까?

CUT는 짝이 없는(unpaired) 학습입니다. "이 SAR의 정답 광학영상은 이것"이라는 픽셀 정답이 없습니다. GAN은 그저 "광학처럼만 보이면" 통과시키기 때문에, G가 그럴듯하게 보이려고 입력에 없던 구조/색을 지어내는 자유를 갖습니다 → 이게 허상입니다. 두 λ는 이 자유에 물리적 제약(규칙)을 더해 허상을 줄입니다.

Edge λ  2. 구조 허상을 누르는 gradient_loss

아이디어는 한 문장입니다 → "출력의 윤곽(에지)은 입력의 윤곽과 같아야 한다." 입력 SAR에 없는 곳에 출력이 에지를 만들면 벌점, 있는 에지를 지워도 벌점. 그래서 G가 멋대로 구조를 못 만듭니다.

연산 4단계 (코드 그대로)

1
밝기(luminance)로 변환 — 컬러(RGB)를 사람이 보는 밝기 한 채널로 합칩니다. 값 범위를 [-1,1] → [0,1] 로 옮긴 뒤 0.299·R + 0.587·G + 0.114·B. 윤곽은 색이 아니라 밝기 변화이므로 이렇게 단순화합니다.
2
입력만 살짝 블러(blur) — SAR에는 스페클(speckle)이라는 알갱이 노이즈가 많습니다. 그 노이즈까지 "윤곽"으로 따라 그리면 안 되므로, 입력을 3×3 평균풀링으로 부드럽게 만들어 진짜 구조만 남깁니다. (출력 fake_B는 블러하지 않습니다.)
3
에지 = 옆 픽셀과의 차이(gradient) — 가로차이 dx = 픽셀[x+1] − 픽셀[x], 세로차이 dy = 픽셀[y+1] − 픽셀[y]. 차이가 크면 그곳이 윤곽입니다. 아래는 한 줄을 스캔하며 옆 칸과 빼서 에지를 뽑는 모습입니다. 에지맵
4
두 에지맵의 차이를 L1로 합산 — 입력 에지맵과 출력 에지맵을 픽셀별로 빼서 절댓값 평균:
Ledge = mean|∂x(real_A') − ∂x(fake_B)| + mean|∂y(real_A') − ∂y(fake_B)|
출력이 입력과 같은 자리에 같은 세기의 윤곽을 가질 때만 0에 가까워집니다. → 입력에 없는 건물 윤곽을 만들면 즉시 벌점 = 구조 허상 억제. 
# models/losses_extra.py
def gradient_loss(source, generated, blur=True):
    gs = _luminance(source)          # 1) 밝기로
    gg = _luminance(generated)
    if blur:
        gs = F.avg_pool2d(gs, 3, stride=1, padding=1)   # 2) 입력만 블러
    s_dx = gs[:,:,:,1:] - gs[:,:,:,:-1]   # 3) 가로 에지
    s_dy = gs[:,:,1:,:] - gs[:,:,:-1,:]   #    세로 에지
    g_dx = gg[:,:,:,1:] - gg[:,:,:,:-1]
    g_dy = gg[:,:,1:,:] - gg[:,:,:-1,:]
    return (s_dx-g_dx).abs().mean() + (s_dy-g_dy).abs().mean()   # 4) L1
⚠️ SAR와 광학은 다른 센서라 밝기 자체는 다를 수 있습니다. 이 손실은 "밝기값"이 아니라 밝기의 변화(윤곽 위치/세기)를 맞추는 구조 보존 규칙입니다. 그래서 모달리티가 달라도 통합니다.

Color λ  3. 색 허상을 누르는 color_moment_loss

아이디어 → "진짜 광학영상을 G에 넣으면 색이 그대로여야 한다." 진짜 광학 real_B 를 G에 통과시킨 결과 idt_B = G(real_B) 의 색 통계가 원본 real_B 와 같아야 합니다. G가 색을 틀어버리는(틴트) 버릇을 막아, fake_B의 색도 현실적이게 만듭니다.

색을 숫자 2개로 요약: 평균(μ)과 표준편차(σ) — "moment"

채널(R,G,B)마다 이미지 전체 픽셀의 평균 μ(전반적인 색조)와 표준편차 σ(대비·색 퍼짐)를 구합니다. 이 둘이 같으면 "색 분위기"가 같다고 봅니다. 아래 애니메이션은 G가 만든 분포(분홍)가 목표(회색)에 평균을 맞추고(좌우 이동) 표준편차를 맞추는(폭 조절) 과정입니다.

목표 분포(real_B) G 출력 분포(idt_B) → 평균·표준편차를 맞춰감
μ
평균 맞추기 — 채널별 평균색이 같아지도록. 다르면 전체가 누렇게/푸르게 뜨는 색 틴트 허상이 생깁니다.
σ
표준편차 맞추기 — 대비/채도 퍼짐이 같아지도록. 너무 밋밋하거나 과채도가 되는 걸 막습니다.
Lcolor = meanc|μc(idt_B) − μc(real_B)| + meanc|σc(idt_B) − σc(real_B)|  (c = R,G,B) 
# models/losses_extra.py
def color_moment_loss(generated, reference):
    g_mean = generated.mean(dim=(2,3))          # 채널별 평균 μ
    r_mean = reference.mean(dim=(2,3))
    g_std  = generated.var(dim=(2,3), unbiased=False).add(EPS).sqrt()   # 채널별 σ
    r_std  = reference.var(dim=(2,3), unbiased=False).add(EPS).sqrt()
    return (g_mean-r_mean).abs().mean() + (g_std-r_std).abs().mean()    # L1
📌 왜 fake_B가 아니라 idt_B로 색을 맞출까요? fake_B는 "SAR→광학"이라 정답 색이 없습니다. 하지만 real_B를 넣은 idt_B는 원본 real_B라는 명확한 색 정답이 있습니다. 여기서 색 규칙을 배우면 G의 색 감각 전체가 교정되어 fake_B에도 전이됩니다. 그래서 nce_idt=True 가 필요합니다.

4. λ 값은 "규칙의 세기" 손잡이

λ는 그 항을 전체 손실에 얼마나 강하게 반영할지 결정합니다. 0이면 끔, 값이 클수록 강하게 강제. 아래는 λ를 0→크게 올릴 때 출력이 선명해지고(에지 일치) 색이 안정되는 모습입니다.


허상 사라짐
단, λ가 너무 크면 GAN이 광학답게 만드는 자유를 과하게 눌러 결과가 흐릿하거나 SAR처럼 밋밋해질 수 있습니다. 적당히가 중요합니다(아래 추천값).

Edge λ = "출력 윤곽을 입력 윤곽에 맞춰라" (입력 real_A ↔ 출력 fake_B 의 그래디언트 L1) → 구조 허상↓

둘 다 λ로 세기를 조절해 생성기 손실에 더해지고, 그만큼 G의 "지어내기" 자유를 제약합니다.
